Example Technical SEO Audit Prompts

Comprehensive Technical SEO Audit Prompt Example

Create a comprehensive technical SEO audit framework for an e-commerce website built on Shopify. The audit should include:

  • A complete technical SEO checklist organized by priority (critical, high, medium, low)
  • Specific tools to use for each audit area with step-by-step instructions
  • Common Shopify-specific technical SEO issues to look for
  • Interpretation guidance for key metrics and findings
  • Remediation recommendations for common issues
  • Implementation prioritization framework based on impact vs. effort

The audit should cover these key areas:

  • Crawlability and indexation (robots.txt, sitemap, crawl stats)
  • Site architecture and internal linking
  • URL structure and canonicalization
  • Page speed and Core Web Vitals
  • Mobile optimization
  • Structured data implementation
  • HTTPS and security configuration
  • JavaScript rendering issues
  • International SEO considerations
  • E-commerce specific factors (faceted navigation, product variants, etc.)

For each area, provide specific examples of what "good" looks like versus common problems, and include both quick wins and long-term strategic recommendations.

Site Speed Optimization Prompt Example

Create a comprehensive site speed audit and optimization plan for a WordPress blog that's experiencing slow loading times. The audit should include:

  • Step-by-step process for measuring current performance using PageSpeed Insights, GTmetrix, and WebPageTest
  • Guidance on interpreting Core Web Vitals metrics (LCP, FID, CLS) and their impact on SEO
  • Common WordPress-specific speed issues to look for (plugin bloat, theme optimization, etc.)
  • Server-side optimization recommendations (hosting, TTFB, caching)
  • Front-end optimization checklist (image optimization, code minification, etc.)
  • Specific WordPress plugin recommendations for different speed optimizations
  • Mobile vs. desktop speed considerations
  • Implementation plan organized by impact level and implementation difficulty

For each recommendation, include:

  • Expected impact on key speed metrics
  • Implementation complexity (easy, moderate, advanced)
  • Whether it requires developer assistance
  • Potential risks or considerations
  • Before/after benchmarking methodology

The goal is to create a practical, actionable plan that can improve site speed metrics to meet Google's Core Web Vitals requirements and enhance user experience.

Crawlability & Indexation Prompt Example

Develop a comprehensive crawlability and indexation audit framework for a large e-commerce site (5,000+ pages) that's experiencing indexing issues. The audit should include:

  • Step-by-step process for analyzing current indexation status using Google Search Console
  • Methodology for identifying crawl budget waste and optimization opportunities
  • Robots.txt analysis and optimization recommendations
  • XML sitemap strategy and implementation best practices
  • Canonical tag audit process and common issues
  • Internal linking architecture assessment
  • Pagination and faceted navigation handling
  • JavaScript rendering and indexation considerations
  • Log file analysis methodology and key metrics to track

The audit should also include:

  • Common indexation issues specific to e-commerce sites
  • Guidance on identifying and fixing "orphaned" pages
  • Recommendations for handling out-of-stock or discontinued products
  • Category page optimization for crawlability
  • Strategies for prioritizing the most important pages for crawling
  • Implementation plan with priority levels and expected outcomes

For each recommendation, provide specific examples of proper implementation and common mistakes to avoid.

Mobile SEO Audit Prompt Example

Create a comprehensive mobile SEO audit framework for a business website that needs to improve its mobile user experience and rankings. The audit should include:

  • Mobile-first indexing compliance checklist
  • Mobile usability testing methodology using Google Search Console and other tools
  • Mobile page speed analysis framework with Core Web Vitals focus
  • Touch element sizing and spacing assessment
  • Mobile navigation and menu structure evaluation
  • Mobile-specific content display issues (hidden content, tabs, accordions)
  • Mobile viewport configuration check
  • Mobile-friendly testing across different devices and screen sizes

The audit should also address:

  • Mobile vs. desktop content parity issues
  • Mobile-specific structured data considerations
  • Local SEO factors for mobile users
  • Mobile form optimization
  • Mobile-specific user intent considerations
  • AMP implementation evaluation (if applicable)
  • Mobile-specific analytics setup and KPIs to track

For each area, provide specific testing methodologies, tools to use, and actionable recommendations with implementation guidance tailored to business websites.

Structured Data Audit Prompt Example

Create a comprehensive structured data audit and implementation plan for a recipe blog. The audit should include:

  • Current structured data assessment methodology
  • Recipe schema.org markup best practices
  • Additional relevant schema types for food blogs (Article, Person, WebSite, etc.)
  • Schema validation testing process
  • Rich result eligibility requirements for recipes
  • Common structured data errors and how to fix them
  • Implementation methods comparison (manual JSON-LD vs. plugins)

The implementation plan should include:

  • Prioritized list of schema types to implement based on SEO impact
  • Required properties for each schema type with examples
  • Optional properties that can enhance rich results
  • Nested schema relationship recommendations
  • Testing and monitoring framework for schema performance
  • Schema update process for content changes
  • Future structured data opportunities and emerging schema types

Include specific examples of properly formatted JSON-LD for recipe content, and guidance on how to track the impact of structured data implementation on search visibility and click-through rates.
